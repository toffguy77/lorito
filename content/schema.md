# Content schema

Source of truth for Lorito's study content. One file per card; the pipeline in
`tools/` validates these and compiles them into the app bundle.

## Layout

```
content/
  themes.json          # per-level theme registry (generated by the pipeline)
  A1/ A2/ B1/ B2/ C1/ C2/
    <id>.md            # one card per file, e.g. A1-15.md
```

A card file lives under the directory matching its `level`.

## Card file format

YAML frontmatter followed by a Markdown body:

```markdown
---
id: A1-15
level: A1
theme: a1-3
order: 15
title: "Глагол ser"
aliases: ["Verbo ser", "быть (ser)"]
related: ["A1-16", "A1-14"]
tags: ["gramática"]
---
> **Суть**
> ...Markdown body...
```

### Frontmatter fields

| Field | Required | Type | Rules |
|-------|----------|------|-------|
| `id` | yes | string | Matches `^[A-C][12]-\d{2}$` (e.g. `A1-07`). Unique across all cards. |
| `level` | yes | string | One of `A1 A2 B1 B2 C1 C2`. Must match the file's directory. |
| `theme` | yes | string | A theme id present in `themes.json` whose level equals this card's level. |
| `order` | yes | int | ≥ 1, unique within the level. Equals the numeric part of `id`. |
| `title` | yes | string | Non-empty. |
| `aliases` | no | string[] | Alternative names (Spanish term, Russian name). |
| `related` | no | string[] | Card ids; each MUST resolve to an existing card. |
| `tags` | no | string[] | Category tags: `gramática`, `vocabulario`, `ortografía`, `fonética`. |

## Levels

Six ordered levels: `A1 < A2 < B1 < B2 < C1 < C2`. Selecting a target level
includes that level and all lower levels.

## Themes

Each level is divided into ordered **themes** (sections). A theme groups a
**contiguous** run of cards by `order` — themes never interleave. The registry
`themes.json` declares every theme:

```json
{ "themes": [ { "id": "a1-1", "level": "A1", "title": "Орфография и фонетика", "order": 1 } ] }
```

Theme ids are `<level-lowercased>-<n>` (e.g. `a1-1`). Titles come from the
source level MOC section headings.

## Body conventions

The Markdown body uses callout blockquotes for the four standard sections,
rendered specially by the app:

- `> **Суть**` — essence / summary
- `> **Ключевые моменты**` — key points
- `> **Частые ошибки**` — common mistakes
- `> **Полезно**` — useful notes

Tables (GFM) are supported. The card `title` is stored in frontmatter, so the
body SHOULD NOT repeat it as a top-level `#` heading.

## Pipeline

See `tools/`:

- `migrate.py` — seed from the Obsidian vault into this format + generate `themes.json`
- `enrich.py` — expand a card body (identity-preserving; LLM integration point)
- `validate.py` — enforce every rule above (non-zero exit on failure)
- `compile.py` — validate, then compile to the app bundle (`content.json`)
- `test_pipeline.py` — pipeline unit tests
